# QuranSays

A Quran-grounded Q&A application. Every answer is sourced exclusively from **The Clear Quran** (Dr. Mustafa Khattab's thematic English translation) with direct ayah citations. Ask any question — Islam has guidance on everything.

---

## Features

- 💬 **Chat** — Ask anything; get answers backed by quoted ayahs with surah/verse references
- 🔍 **Clarification** — Ambiguous questions trigger follow-up questions before answering
- 📖 **Verse pages** — `/{surah}/{ayah}` shows the ayah text with contextual explanation
- ⚡ **Caching** — Repeated questions are served instantly from an in-memory cache
- 🔒 **Source policy** — The LLM is strictly grounded; no answer is given without Quranic evidence

## First-Time Setup

### 1. Clone the repo

```bash
git clone https://github.com/Noor-e-Bilal/QuranSays.git
cd QuranSays
```

### 2. Configure your API key

```bash
cp .env.example apps/web/.env.local
```

Open `apps/web/.env.local` and replace `your_api_key_here` with your **opencode.ai** API key:

```env
ANTHROPIC_BASE_URL=https://opencode.ai/zen
ANTHROPIC_MODEL=minimax-m2.5-free
ANTHROPIC_API_KEY=<your-opencode-api-key>

CHROMA_URL=http://localhost:8000
DB_PATH=../../data/quran.db
```

> Get your key at [opencode.ai](https://opencode.ai) → Dashboard → API Keys.

### 3. Install dependencies

```bash
make install
```

This installs Python packages (`pdfplumber`, `chromadb`, etc.) and runs `npm install` inside `apps/web`.

### 4. Place the source PDF

Put the PDF in the `Docs/` folder (already present in the repo):

```
Docs/the-clear-quran-a-thematic-english-translation.pdf
```

> The PDF is not committed to the repo for copyright reasons. Contact the maintainer if you need it.

### 5. Run the ingestion pipeline

This extracts all 6,236 ayahs from the PDF, builds the SQLite database, and creates ChromaDB vector embeddings. **Run once; takes ~5 minutes.**

```bash
# Start ChromaDB first
docker compose up -d chroma

# Then ingest
make ingest
```

> You can verify ingestion succeeded:
> ```bash
> python3 -c "import sqlite3; c=sqlite3.connect('data/quran.db'); print(c.execute('SELECT COUNT(*) FROM quran_ayah').fetchone())"
> # Expected: (6236,)
> ```

## How It Works

1. **Ingestion** — The PDF is parsed page-by-page; ayahs are extracted and stored in SQLite with an FTS5 full-text index. Sentence embeddings are generated and stored in ChromaDB.

2. **Retrieval** — When a question arrives, the app runs a hybrid search: FTS keyword matching + ChromaDB semantic similarity. Both scores are merged (40% FTS / 60% semantic) and the top 10 ayahs are selected as evidence.

3. **Clarification** — If the LLM detects the question is vague or ambiguous, it returns 2-3 specific follow-up options instead of an answer. The user's clarification is combined with the original question and re-submitted.

4. **Generation** — The top-ranked ayahs are injected into the LLM prompt. The model is instructed to answer strictly from the provided evidence and cite specific ayahs.

5. **Caching** — Completed answers are cached for 5 minutes so repeated questions return in <100ms.
